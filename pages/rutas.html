1) ¿Qué es React Router y por qué usarlo?

React Router nos permite crear “páginas” dentro de una app de React (Home, Sobre mí, Contacto…), navegando entre ellas sin recargar el navegador. Ideal para sitios con varias secciones.

2) Instalación

npm i react-router-dom

Tras instalar, reinicia si fuera necesario npm run dev.

3) Estructura mínima de archivos

src/
├─ main.jsx
├─ App.jsx
├─ pages/
│  ├─ Home.jsx
│  ├─ Sobre.jsx
│  └─ Contacto.jsx
└─ components/
   └─ Navbar.jsx

4) Navbar con enlaces (Link/NavLink)

// src/components/Navbar.jsx
import { NavLink } from "react-router-dom";

export default function Navbar(){
  const active = ({ isActive }) => ({
    textDecoration: isActive ? "underline" : "none",
    fontWeight: isActive ? 700 : 600
  });

  return (
    <nav className="mt-4">
      <ul style={{ display:"flex", gap:"1rem", listStyle:"none", padding:0 }}>
        <li><NavLink style={active} to="/" end>Inicio</NavLink></li>
        <li><NavLink style={active} to="/sobre">Sobre mí</NavLink></li>
        <li><NavLink style={active} to="/contacto">Contacto</NavLink></li>
      </ul>
    </nav>
  );
}

6) Definir rutas

// src/App.jsx
import { Routes, Route } from "react-router-dom";
import Navbar from "./components/Navbar";
import Home from "./pages/Home";
import Sobre from "./pages/Sobre";
import Contacto from "./pages/Contacto";

function NotFound(){
  return <p className="mt-2">404 · Página no encontrada</p>;
}

export default function App(){
  return (
    <>
      <header className="react-header">
        <div className="react-container">
          <h1 className="react-title">Mi sitio con rutas</h1>
          <p className="react-subtitle">React + React Router</p>
        </div>
      </header>

      <main className="react-container">
        <Navbar />

        <Routes>
          <Route path="/" element={<Home />} />
          <Route path="/sobre" element={<Sobre />} />
          <Route path="/contacto" element={<Contacto />} />
          <Route path="*" element={<NotFound />} />
        </Routes>
      </main>
    </>
  );
}

7) Activar el router en main.jsx

// src/main.jsx
import React from "react";
import ReactDOM from "react-dom/client";
import { BrowserRouter } from "react-router-dom";
import App from "./App.jsx";
import "./styles/react-theme.css"; // o tu css/style.css

ReactDOM.createRoot(document.getElementById("root")).render(
  <React.StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </React.StrictMode>
);

Listo: ya puedes navegar entre /, /sobre y /contacto sin recargar la página.

8) Bonus (opcional): Rutas anidadas

// src/pages/Blog.jsx
import { Outlet, NavLink } from "react-router-dom";
export default function Blog(){
  return (
    <section className="card">
      <h2>📝 Blog</h2>
      <nav className="mt-2">
        <NavLink to="articulo-1">Artículo 1</NavLink> ·{" "}
        <NavLink to="articulo-2">Artículo 2</NavLink>
      </nav>
      <Outlet />
    </section>
  );
}

// src/App.jsx (añade)
// <Route path="/blog" element={<Blog />}>

//   <Route path="articulo-2" element={<p>Contenido 2</p>} />

Las anidadas sirven para secciones con subpáginas (catálogos, paneles, etc.).

9) Publicar con rutas

  • En Vercel/Netlify, el comportamiento por defecto suele funcionar con BrowserRouter si hay “fallback” al index.html (en Netlify: regla de reescritura /* /index.html 200).
  • Alternativa sencilla: usar <HashRouter> en vez de BrowserRouter; evita configurar rewrites pero las URLs incluyen #.
