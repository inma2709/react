🎯 ¿Qué son las Props?

Las Props son como los "ingredientes" que le das a una receta. Imagínate que tienes una receta para hacer una tarjeta de presentación, pero quieres que cada tarjeta sea diferente.

🍪 Analogía del mundo real:

🏪 Imagina una fábrica de tarjetas:

  • 📐 Molde (Componente): La máquina que hace tarjetas
  • 🎨 Ingredientes (Props): Nombre, color, foto, título...
  • 📇 Resultado: Tarjetas únicas con diferentes datos

💡 La clave: Un mismo molde puede crear infinitas tarjetas diferentes según los ingredientes que le pongas.

✨ En React funciona igual:

  • 🧩 Componente: La "plantilla" reutilizable
  • 📦 Props: Los datos específicos que le pasas
  • ⚛️ React: Combina componente + props = UI única

🎁 Props: Haciendo Componentes Inteligentes

🤔 ¿Recuerdas las etiquetas HTML con atributos?

En HTML puedes escribir:

<img src="mi-foto.jpg" alt="Mi foto" width="200" />

Los atributos (src, alt, width) hacen que la misma etiqueta img pueda mostrar diferentes imágenes.

🎯 ¡Las Props son lo mismo pero para tus componentes!

Las props son "atributos personalizados" que le pasas a un componente como parametros para que sea reutilizable.

🛠️ Ejemplo práctico: Creemos un componente Tarjeta profesional

Vamos a crear una tarjeta moderna con CSS separado, imagen opcional y botón. Crea dos archivos:

📄 Código del componente Tarjeta.jsx (Click para ver/ocultar) 
import './Tarjeta.css';

function Tarjeta({ titulo, descripcion, imagen, textoBoton, color, colorFondo }) {
  return (
    <div 
      className="tarjeta" 
      style={{ 
        borderLeftColor: color,
        backgroundColor: colorFondo || '#ffffff'
      }}
    >
      {imagen && (
        <div className="tarjeta-imagen">
          <img src={imagen} alt={titulo} />
        </div>
      )}
      
      <div className="tarjeta-contenido">
        <h3 className="tarjeta-titulo">{titulo}</h3>
        <p className="tarjeta-descripcion">{descripcion}</p>
        
        {textoBoton && (
          <button className="tarjeta-boton" style={{ backgroundColor: color }}>
            {textoBoton}
          </button>
        )}
      </div>
    </div>
  );
}

export default Tarjeta;

🎯 ¿Qué mejoras tiene la nueva Tarjeta?

💡 ¿Por qué separar CSS del JSX?
  • Organización: Estilos en un archivo, lógica en otro
  • Reutilización: Los estilos se pueden aplicar a múltiples elementos
  • Mantenimiento: Más fácil modificar el diseño
  • Performance: CSS se cachea por el navegador
  • Responsividad: Media queries para diferentes pantallas

  • Importamos los estilos CSS desde el archivo en la misma carpeta
  • React aplicará automáticamente estas clases cuando el componente se renderice

  • Ahora recibimos 5 props en lugar de 3

�️ Línea 5: <div className="tarjeta" style={{ borderLeftColor: color }}>

  • className="tarjeta" aplica los estilos base del CSS

  • className="tarjeta-contenido" - Aplica estilos flex para organizar el contenido

  • {textoBoton && (...)} - Solo aparece si se pasa textoBoton
  • style={{ backgroundColor: color }} - El botón toma el color de la prop
  • Combina estilos CSS base con color personalizado

🤔 Respuesta corta: En nuestro componente actual son OPCIONALES pero PROBLEMÁTICAS si no las pasas

❌ ¿Qué pasa si no pasas las props?
// Si usas el componente sin props:
<Tarjeta />

// Resultado:
// - titulo será undefined → <h3>undefined</h3>
// - descripcion será undefined → <p>undefined</p>  
// - color será undefined → backgroundColor: undefined (transparente)
🚨 Problemas que verás:
  • La tarjeta aparece sin color de fondo
  • Muestra literalmente "undefined" en pantalla
  • Se ve rota y poco profesional
✅ Solución 1: Props con valores por defecto
function Tarjeta({ titulo = "Sin título", descripcion = "Sin descripción", color = "#cccccc" }) {
  return (
    <div style={{
      backgroundColor: color,
      padding: "15px",
      borderRadius: "8px", 
      margin: "10px 0",
      color: "white"
    }}>
      <h3>{titulo}</h3>
      <p>{descripcion}</p>
    </div>
  );
}
✅ Solución 2: Validación con OR (||)
function Tarjeta({ titulo, descripcion, color }) {
  return (
    <div style={{
      backgroundColor: color || "#cccccc",  // Si no hay color, usa gris
      padding: "15px",
      borderRadius: "8px",
      margin: "10px 0", 
      color: "white"
    }}>
      <h3>{titulo || "Sin título"}</h3>
      <p>{descripcion || "Sin descripción"}</p>
    </div>
  );
}
🎯 ¿Cuándo usar cada enfoque?
  • Valores por defecto: Cuando quieres que el componente funcione bien sin props
  • Sin valores por defecto: Cuando SIEMPRE esperas que se pasen las props
  • Validación OR: Para casos específicos donde algunos valores pueden faltar
📚 Conceptos importantes:
Situación ¿Qué pasa? ¿Cómo solucionarlo?
<Tarjeta /> Muestra "undefined" Valores por defecto
<Tarjeta titulo="HTML" /> Solo titulo funciona Valores por defecto para las otras
Todas las props pasadas ¡Funciona perfecto! Nada, así está bien
🏫 Para el aula: ¿Cómo explicarlo a los estudiantes?

"Las props en React son como los ingredientes de una receta:

  • Puedes cocinar sin algunos ingredientes, pero el plato no sabrá igual
  • Es mejor tener ingredientes de repuesto (valores por defecto) por si faltan
  • O hacer que ciertos ingredientes sean obligatorios para que el plato salga bien"

🎯 El poder de la reutilización: Un componente, infinitas posibilidades

En Home.jsx estamos viendo algo increíble: el mismo componente Tarjeta usado 6 veces, pero cada una se ve completamente diferente. ¿Cómo es posible?

🔥 1. Dos filas organizadas estratégicamente:
🖼️ Primera fila - CON IMÁGENES:
  • Tarjeta 1: Completa (imagen + botón + fondo azul)
  • Tarjeta 2: Imagen + fondo degradado (sin botón)
  • Tarjeta 3: Imagen + botón + fondo amarillo
📝 Segunda fila - SIN IMÁGENES:
  • Tarjeta 4: Minimalista (solo texto, fondo blanco)
  • Tarjeta 5: Con botón + fondo verde
  • Tarjeta 6: Con botón + fondo morado
🎨 2. Variedad de fondos y colores:
// Fondo personalizado sólido:
colorFondo="#f0f8ff"

// Fondo degradado CSS:
colorFondo="linear-gradient(135deg, #fff3e0 0%, #ffccbc 100%)"

// Sin fondo (blanco por defecto):
{/* No se pasa la prop colorFondo */}
🖼️ 3. Imágenes dinámicas de Unsplash:
// Imágenes reales y responsivas:
imagen="https://images.unsplash.com/photo-1633356122544-f134324a6cee?w=150&h=150&fit=crop"
💡 ¿Qué está pasando aquí?
  1. React ve: "Ah, necesito crear 6 componentes Tarjeta diferentes"
  2. Por cada uno: Llama a la función Tarjeta() con props únicas
  3. Resultado: 6 tarjetas completamente diferentes del mismo componente
  4. Layout: Organizadas en 2 filas horizontales de 3 tarjetas cada una
🎨 4. Personalización total sin límites:

Cada tarjeta es independiente. Mira la variedad que conseguimos:

  • Bootcamp: Con botón + fondo morado

"Props = Personalización infinita sin duplicación"

  • Un componente se escribe una vez
  • Se usa infinitas veces con props diferentes
  • Cada uso es único gracias a las props opcionales
  • 6 tarjetas diferentes = 6 combinaciones del mismo código
  • Cambios en el componente afectan todas las instancias

Así, con un solo componente Tarjeta y 6 props diferentes, creamos una página dinámica y visualmente atractiva. Header y Footer se muestran en todas las páginas, y App se encarga solo de las rutas.
