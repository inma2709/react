🎯 ¿Qué son las Props?

Las Props son como los "ingredientes" que le das a una receta. Imagínate que tienes una receta para hacer una tarjeta de presentación, pero quieres que cada tarjeta sea diferente.

🍪 Analogía del mundo real:

🏪 Imagina una fábrica de tarjetas:

  • 📐 Molde (Componente): La máquina que hace tarjetas
  • 🎨 Ingredientes (Props): Nombre, color, foto, título...
  • 📇 Resultado: Tarjetas únicas con diferentes datos

💡 La clave: Un mismo molde puede crear infinitas tarjetas diferentes según los ingredientes que le pongas.

✨ En React funciona igual:

  • 🧩 Componente: La "plantilla" reutilizable
  • 📦 Props: Los datos específicos que le pasas
  • ⚛️ React: Combina componente + props = UI única

🎁 Props: Haciendo Componentes Inteligentes

🤔 ¿Recuerdas las etiquetas HTML con atributos?

En HTML puedes escribir:

<img src="mi-foto.jpg" alt="Mi foto" width="200" />

Los atributos (src, alt, width) hacen que la misma etiqueta img pueda mostrar diferentes imágenes.

🎯 ¡Las Props son lo mismo pero para tus componentes!

Las props son "atributos personalizados" que le pasas a un componente como parametros para que sea reutilizable pero no siempre igual.

🛠️ Ejemplo práctico: Creemos un componente Tarjeta profesional

Vamos a crear una tarjeta moderna con CSS separado, imagen opcional y botón. Crea dos archivos:

📄 Código del componente Tarjeta.jsx (Click para ver/ocultar) 
import './Tarjeta.css';

    // Componente Tarjeta: recibe props para personalizar cada tarjeta
    function Tarjeta({ titulo, descripcion, imagen, textoBoton, color, colorFondo }) {
      return (
        // Contenedor principal de la tarjeta
        // borderLeftColor y backgroundColor se personalizan con props
        <div 
      className="tarjeta" 
      style={{ 
        borderLeftColor: color,              // Color del borde izquierdo
        backgroundColor: colorFondo || '#ffffff' // Fondo personalizado o blanco por defecto
      }}
        >
      {/* Renderiza la imagen solo si la prop 'imagen' existe */}
      {imagen && (
        <div className="tarjeta-imagen">
          <img src={imagen} alt={titulo} /> {/* alt mejora accesibilidad */}
        </div>
      )}
      
      {/* Contenido principal de la tarjeta */}
      <div className="tarjeta-contenido">
        <h3 className="tarjeta-titulo">{titulo}</h3> {/* Título dinámico */}
        <p className="tarjeta-descripcion">{descripcion}</p> {/* Descripción dinámica */}
        
        {/* Renderiza el botón solo si la prop 'textoBoton' existe */}
        {textoBoton && (
          <button 
        className="tarjeta-boton" 
        style={{ backgroundColor: color }} // Color del botón igual al borde
          >
        {textoBoton} {/* Texto del botón personalizado */}
          </button>
        )}
      </div>
        </div>
      );
    }

    export default Tarjeta; // Exporta el componente para usarlo en otras partes

❌ ¿Qué pasa si no pasas las props?
// Si usas el componente sin props:

// Resultado:
// - titulo será undefined → <h3>undefined</h3>
// - descripcion será undefined → <p>undefined</p>  
// - color será undefined → backgroundColor: undefined (transparente)
🚨 Problemas que verás:
  • La tarjeta aparece sin color de fondo
  • Muestra literalmente "undefined" en pantalla
  • Se ve rota y poco profesional
✅ Solución 1: Props con valores por defecto
function Tarjeta({ titulo = "Sin título", descripcion = "Sin descripción", color = "#cccccc" }) {
  return (
    <div style={{
      backgroundColor: color,
      padding: "15px",
      borderRadius: "8px", 
      margin: "10px 0",
      color: "white"
    }}>
      <h3>{titulo}</h3>
      <p>{descripcion}</p>
    </div>
  );
}
✅ Solución 2: Validación con OR (||)
function Tarjeta({ titulo, descripcion, color }) {
  return (
    <div style={{
      backgroundColor: color || "#cccccc",  // Si no hay color, usa gris
      padding: "15px",
      borderRadius: "8px",
      margin: "10px 0", 
      color: "white"
    }}>
      <h3>{titulo || "Sin título"}</h3>
      <p>{descripcion || "Sin descripción"}</p>
    </div>
  );
}
🎯 ¿Cuándo usar cada enfoque?
  • Valores por defecto: Cuando quieres que el componente funcione bien sin props
  • Sin valores por defecto: Cuando SIEMPRE esperas que se pasen las props
  • Validación OR: Para casos específicos donde algunos valores pueden faltar
📚 Conceptos importantes:
Situación ¿Qué pasa? ¿Cómo solucionarlo?
<Tarjeta /> Muestra "undefined" Valores por defecto
<Tarjeta titulo="HTML" /> Solo titulo funciona Valores por defecto para las otras
Todas las props pasadas ¡Funciona perfecto! Nada, así está bien
🏫 Resumen ¿Qué son los props?

"Las props en React son como los ingredientes de una receta:

  • Puedes cocinar sin algunos ingredientes, pero el plato no sabrá igual
  • Es mejor tener ingredientes de repuesto (valores por defecto) por si faltan
  • O hacer que ciertos ingredientes sean obligatorios para que el plato salga bien"

🧠 Conectando con lo que ya sabes: Arrays y forEach

Antes de React, ya sabías trabajar con arrays:

📚 Repaso rápido de arrays básicos:

// Array que ya conoces
const colores = ["rojo", "azul", "verde", "amarillo"];

// forEach (que ya dominas) - Ejecuta algo con cada elemento
colores.forEach(color => {
  console.log("Me gusta el color", color);
});

// Resultado en consola:
// Me gusta el color rojo
// Me gusta el color azul  
// Me gusta el color verde
// Me gusta el color amarillo

🔄 ¿Qué hace .map()? ¡Es similar pero diferente!

// .map() - TRANSFORMA cada elemento y crea un nuevo array
const coloresEnMayuscula = colores.map(color => {
  return color.toUpperCase();
});

console.log(coloresEnMayuscula);
// Resultado: ["ROJO", "AZUL", "VERDE", "AMARILLO"]

⚡ Diferencia clave entre forEach y map:

Método ¿Qué hace? ¿Devuelve algo? Uso típico
forEach Ejecuta una función por cada elemento ❌ No (undefined) Mostrar, imprimir, efectos secundarios
map Transforma cada elemento ✅ Sí (nuevo array) Crear, transformar, generar

🎯 ¿Por qué .map() es perfecto para React?

Porque React necesita que le "devuelvas" elementos JSX, no que imprimas en consola:

❌ forEach NO funciona en React:
// ❌ ESTO NO FUNCIONA - forEach no devuelve nada
{colores.forEach(color => (
  <div>{color}</div>  // React no recibe estos elementos
))}

// Resultado: No se muestra nada
✅ map SÍ funciona en React:
// ✅ ESTO SÍ FUNCIONA - map devuelve un array de elementos JSX
{colores.map(color => (
  <div key={color}>{color}</div>  // React recibe un array de elementos
))}

// Resultado: 4 divs con los colores

🏭 Analogía perfecta: La fábrica de productos

Imagina una fábrica:

  • 🏭 forEach: "Inspecciona cada producto que pasa por la cinta" - Solo mira, no crea nada nuevo
  • 🎯 map: "Por cada material que entra, produce un producto terminado" - Transforma y crea
  • ⚛️ En React: Por cada dato del array, crea un componente visual

💡 Demostración práctica con nombres:

// Datos simples
const nombres = ["Ana", "Luis", "María", "Carlos"];

// 1. forEach - Solo hacer algo (no crear)
nombres.forEach(nombre => {
  console.log(`Hola ${nombre}`);  // Solo imprime
});

// 2. map - Crear algo nuevo  
const saludos = nombres.map(nombre => {
  return `Hola ${nombre}`;  // Crea un nuevo array de saludos
});

console.log(saludos);
// ["Hola Ana", "Hola Luis", "Hola María", "Hola Carlos"]

// 3. En React - map crea componentes
const componentesSaludo = nombres.map(nombre => (
  <div key={nombre}>Hola {nombre}</div>
));

🎓 Para recordar :

  • forEach: "Haz algo con cada elemento" (efecto secundario)
  • map: "Convierte cada elemento en algo nuevo" (transformación)
  • React necesita transformación: datos → componentes visuales
  • Por eso usamos map: array de datos → array de componentes JSX

🧠 ¿De dónde sale la sintaxis element={<Proyectos />}?

React Router usa una sintaxis específica para definir qué componente mostrar en cada ruta:

  • path="/" → Define la URL de la ruta
  • element={<Componente />} → Define QUÉ componente React mostrar
📝 Ejemplos de sintaxis:
// ✅ CORRECTO - Componente React
<Route path="/proyectos" element={<Proyectos />} />

// ❌ INCORRECTO - Falta las llaves {}
<Route path="/proyectos" element=<Proyectos /> />

🔍 ¿Por qué element y no component?

React Router v6 cambió de component a element porque:

  • Más flexible: Puedes pasar props directamente
  • Mejor rendimiento: No recrea el componente en cada render
  • JSX nativo: Usas la sintaxis normal de React

🧠 ¿Por qué necesitamos la prop `key`? (Click para entender)

🔑 La prop `key` explicada fácil

Cuando usas .map() en React, SIEMPRE necesitas una prop key única para cada elemento.

🚗 Analogía perfecta: Matrículas de autos

Imagina un parking con 100 autos idénticos:

  • 🚗 Sin matrícula: ¿Cómo distingues cuál es cuál?
  • 🆔 Con matrícula: Cada auto tiene una ID única
  • ⚛️ En React: La prop key es como la matrícula
⚠️ ¿Qué pasa si no pones key?
// ❌ MAL - Sin key (React se queja en consola)
{productos.map(producto => (
  <Tarjeta titulo={producto.titulo} />  {/* Falta key */}
))}

// ✅ BIEN - Con key única
{productos.map(producto => (
  <Tarjeta 
    key={producto.id}              {/* 👈 Key única obligatoria */}
    titulo={producto.titulo} 
  />
))}
📋 Reglas de la prop key:
✅ Buenas prácticas ❌ Evitar
key={producto.id} key={index} (en la mayoría de casos)
key={usuario.email} Sin key
key={item.codigo} key={"fijo"} (misma key repetida)
💡 ¿Por qué React necesita esto?

Optimización de rendimiento:

  • React usa la key para saber qué elementos cambiar, añadir o quitar
  • Sin key, React tiene que regenerar todo cada vez
  • Con key, React solo cambia lo que realmente cambió
  • Resultado: Tu app va más rápida y fluida

❌ Desventajas:
  • 📝 Más código: Requiere import para cada imagen
  • 🔄 Rebuild: Cambiar imagen requiere recompilar
  • 🧠 Complejidad: URLs generadas automáticamente (largas y raras)
  • 📋 Más gestión: Arrays dinámicos son más complicados
🎯 Ideal para: Imágenes críticas, apps con pocas imágenes, máximo rendimiento
☁️ Opción 3: URLs externas (Unsplash, CDN)
🌐 Ejemplo del código:
{
  id: 2,
  titulo: "🚀 James Webb descubre galaxias imposibles",
  imagen: "https://images.unsplash.com/photo-1446776653964-20c1d3a81b06?w=150&h=150&fit=crop",
  // ...
}
✅ Ventajas:
  • 🚀 Velocidad de desarrollo: No necesitas descargar/gestionar archivos
  • 📦 Bundle ligero: Las imágenes no aumentan el tamaño de tu app
  • 🔄 Actualizaciones dinámicas: Cambias URL y ya está
  • 🌍 CDN global: Se sirven desde el servidor más cercano
  • 🎨 Unsplash gratis: Millones de imágenes profesionales
❌ Desventajas:
  • 🌐 Dependencia externa: Si Unsplash cae, tus imágenes no cargan
  • ⚡ Latencia: Conexión a servidor externo
  • 💰 Límites: Algunos CDNs tienen restricciones de uso
  • 🎨 Menos control: No puedes modificar las imágenes
  • 📱 Offline: No funcionan sin internet
🎯 Ideal para: Prototipado rápido, MVPs, demos, portfolios personales
