🧩 Componentes: Las Piezas de LEGO de tu Aplicación

🤔 ¿Qué es un componente?

Imagina que estás construyendo con LEGO. En lugar de tener una pieza gigante para toda la casa, tienes:

  • 🧱 Una pieza para la puerta
  • 🧱 Una pieza para la ventana
  • 🧱 Una pieza para el techo

Los componentes de React son igual: pequeñas piezas reutilizables que juntas forman tu aplicación.

📝 Anatomía de un Componente

Un componente es simplemente una función de JavaScript que devuelve JSX. ¡Así de simple!

🔬 Diseccionemos un componente: 
function MiComponente() {    ← 1) Es una función
  const mensaje = "¡Hola!";  ← 2) Puede tener lógica JS
  
  return (                   ← 3) Devuelve JSX
    <h1>{mensaje}</h1>
  );
}

⚠️ Reglas importantes de los componentes:

  • Nombre en Mayúscula: MiComponente, no miComponente
  • Siempre devuelve JSX (o null si no quieres mostrar nada)
  • Un archivo = Un componente (buena práctica)

👷‍♀️ ¡Manos a la obra! Tu Primer Componente

Vamos a incorporar tu primer componente paso a paso. ¡Sígueme y hazlo en tu proyecto!

🗂️ Paso 1: Crear la carpeta (¡organización es clave!)

  1. En tu proyecto, dentro de src/, crea una carpeta llamada components
  2. Tu estructura debería verse así:
src/
├── components/     ← nueva carpeta
├── App.jsx
└── main.jsx

📄 Paso 2: Crear tu primer archivo de componente

  1. Dentro de src/components/, crea un archivo llamado Header.jsx
  2. A continuación crea otro archivo para los estilos de Header: Header.css
  3. Un componente en React se escribe como una función de JavaScript que devuelve JSX. El nombre del componente debe empezar en mayúscula y el archivo suele tener el mismo nombre que el componente.
  4. Escribe exactamente esto :
📄 Código del componente Header.jsx (Click para ver/ocultar) 
import { Link } from 'react-router-dom';
import './Header.css';

function Header() {
  return (
    <header className="header">
      <h1 className="title">Mi Portafolio</h1>
      <nav className="nav">
        <Link to="/" className="navLink">Inicio</Link>
        <Link to="/proyectos" className="navLink">Proyectos</Link>
        <Link to="/sobre-mi" className="navLink">Sobre mí</Link>
        <button className="button">Contacto</button>
      </nav>
    </header>
  );
}

export default Header;

🔍 Explicación detallada línea por línea (Click para ver/ocultar)

📦 Línea 1: import { Link } from 'react-router-dom';

  • Importamos el componente Link desde la librería react-router-dom
  • Link es como un <a> pero para aplicaciones SPA (Single Page Applications)
  • Permite navegar entre páginas sin recargar toda la aplicación
  • Las llaves { } indican que es una importación específica (destructuring)

🎨 Línea 2: import styles from './Header.css';

  • Importamos los estilos CSS desde el archivo Header.css en la misma carpeta
  • styles será un objeto que contiene todas las clases CSS como propiedades
  • El ./ indica que está en la misma carpeta que el componente
  • Esto se llama "CSS Modules" - cada componente tiene sus propios estilos

⚡ Línea 4: function Header() {

  • Declaramos una función llamada Header (en mayúscula porque es un componente)
  • Es una función flecha o function declaration que define nuestro componente
  • Los componentes siempre deben empezar con mayúscula en React

🔄 Línea 5: return (

  • Todo componente debe return JSX (JavaScript XML)
  • Los paréntesis permiten escribir JSX en múltiples líneas de forma legible

🏗️ Línea 6: <header className={styles.header}>

  • <header> es un elemento HTML semántico para la cabecera
  • className (no class) asigna estilos CSS en JSX
  • {styles.header} accede a la clase CSS .header del archivo importado
  • Las llaves { } permiten ejecutar JavaScript dentro del JSX

📰 Línea 7: <h1 className={styles.title}>Mi Portafolio</h1>

  • Elemento <h1> para el título principal de la página
  • Aplica los estilos de la clase .title del CSS importado
  • "Mi Portafolio" es el texto que aparecerá en la página

🧭 Línea 8: <nav className={styles.nav}>

  • <nav> es el elemento semántico HTML para navegación
  • Aplica los estilos de navegación desde el CSS importado
  • Contiene todos los enlaces de navegación

🔗 Líneas 9-11: Los componentes <Link>

  • <Link to="/">: Navega a la ruta raíz (página de inicio)
  • <Link to="/proyectos">: Navega a la página de proyectos
  • <Link to="/sobre-mi">: Navega a la página "sobre mí"
  • className={styles.navLink}: Aplica estilos específicos a los enlaces
  • Cada Link renderiza como un <a> pero con funcionalidad SPA

🔘 Línea 12: <button className={styles.button}>Contacto</button>

  • Un botón normal de HTML con estilos aplicados
  • No navega automáticamente - necesitarías agregar un onClick para funcionalidad

🏁 Línea 16: export default Header;

  • export default permite que otros archivos importen este componente
  • Es la exportación por defecto, así que se puede importar como import Header from './Header'
  • Sin esto, el componente no sería accesible desde otros archivos

🔧 Penúltimo paso: Actualizar App.jsx

¡Perfecto! Ya tienes tu componente Home funcionando. Ahora necesitas actualizar App.jsx para que use React Router y permita la navegación entre páginas.

🗺️ Código completo de App.jsx con rutas (Click para ver/ocultar) 
import { Routes, Route } from "react-router-dom";
import Home from "./pages/Home";

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/proyectos" element={<div style={{paddingTop: '100px', padding: '100px 2rem'}}>Página de Proyectos (En construcción)</div>} />
      <Route path="/sobre-mi" element={<div style={{paddingTop: '100px', padding: '100px 2rem'}}>Página Sobre Mí (En construcción)</div>} />
    </Routes>
  );
}

export default App;
🧠 Explicación línea por línea de App.jsx (Click para ver/ocultar)

📚 ¿Qué hace cada línea?

🔗 Línea 1: import { Routes, Route } from "react-router-dom";
  • Routes: Contenedor principal que maneja todas las rutas de la aplicación
  • Route: Define una ruta individual (URL específica)
  • react-router-dom: Librería que permite navegación sin recargar página (SPA)
  • Destructuring: Las llaves { } importan componentes específicos
📁 Línea 2: import Home from "./pages/Home";
  • Importación: Trae nuestro componente Home desde la carpeta pages
  • Ruta relativa: ./pages/Home busca en carpeta pages desde App.jsx
  • Organización: Separamos páginas en carpeta específica para mejor estructura
⚡ Líneas 4-5: function App() { return (
  • Componente principal: App es el componente raíz de toda la aplicación
  • Return JSX: Devuelve la estructura de rutas
  • Responsabilidad única: App solo se encarga de definir las rutas
🗺️ Línea 6: <Routes>
  • Contenedor de rutas: Agrupa todas las rutas posibles
  • Router context: Proporciona contexto de navegación a las rutas hijas
  • Solo un Routes: Debe haber un solo Routes activo por aplicación
🏠 Línea 7: <Route path="/" element={<Home />} />
  • path="/": Define la URL raíz (ejemplo.com/)
  • element={<Home />}: Qué componente mostrar en esa ruta
  • Página principal: Primera página que ven los usuarios
🔧 Líneas 8-9: Rutas temporales
  • path="/proyectos": URL para página de proyectos
  • path="/sobre-mi": URL para página sobre mí
  • div temporal: Marcadores mientras desarrollas las páginas reales
  • Estilos inline: paddingTop compensa el header fijo
🎯 ¿Por qué esta estructura?
  • Escalabilidad: Fácil agregar nuevas páginas
  • Navegación funcional: Los enlaces del Header ahora funcionan
  • SPA real: Cambio de páginas sin recargar
  • Desarrollo progresivo: Páginas temporales mientras construyes

📋 Último paso: Actualizar main.jsx

Para que las rutas funcionen, también necesitas actualizar main.jsx. Este archivo es súper importante porque es el punto de entrada de toda tu aplicación React.

🔧 Código actualizado de main.jsx (Click para ver/ocultar) 
import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import { BrowserRouter } from 'react-router-dom'
import App from './App.jsx'

createRoot(document.getElementById('root')).render(
  <StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </StrictMode>,
)
🔧 ¿Por qué necesitamos BrowserRouter en main.jsx? (Click para ver/ocultar)

🤔 La pregunta crucial: ¿Por qué BrowserRouter en main.jsx y no en App.jsx?

🎯 Respuesta corta: Porque BrowserRouter debe envolver TODA la aplicación desde el punto más alto para que las rutas funcionen en cualquier parte.

🏗️ Arquitectura de una aplicación React con rutas:
main.jsx (punto de entrada)
├── StrictMode (modo desarrollo)
├── BrowserRouter (habilita rutas en TODA la app)
└── App.jsx (componente principal)
    └── Routes (define rutas específicas)
        ├── Route (página 1)
        ├── Route (página 2)
        └── Route (página 3)
⚡ ¿Qué pasaría si ponemos BrowserRouter en App.jsx?
  • ❌ Error común: "useRoutes() may be used only in the context of a <Router> component"
  • 🔧 Problema: React Router necesita "contexto" desde el nivel más alto
  • 💡 Solución: BrowserRouter en main.jsx proporciona contexto a toda la aplicación
🎯 ¿Cómo afecta esto a tu día a día como desarrollador?
📅 Configuración inicial (una sola vez):
  • main.jsx: Envuelves la app con BrowserRouter
  • App.jsx: Defines las rutas con Routes y Route
  • Componentes: Usan Link para navegar
🔄 Desarrollo diario (99% del tiempo):
  • Agregar nuevas páginas: Solo modificas App.jsx
  • main.jsx: Lo dejas intacto
  • Navegación: Usas Link en cualquier componente
🧠 Analogía para entenderlo mejor:

🏢 Piensa en un edificio:

  • main.jsx = Cimientos: Proporcionan base sólida para todo el edificio
  • BrowserRouter = Sistema eléctrico: Debe estar desde la base para llegar a todos los pisos
  • App.jsx = Planos de distribución: Definen qué hay en cada piso
  • Componentes = Habitaciones: Pueden usar la electricidad sin instalarla
🚨 Error típico y su solución:

❌ Si no pones BrowserRouter en main.jsx:

// En App.jsx
<Routes>  ← ¡ERROR! No hay contexto de router
  <Route path="/" element={<Home />} />
</Routes>

💥 Error que verás: "useRoutes() may be used only in the context of a <Router> component"

✅ Con BrowserRouter en main.jsx:

// main.jsx
<BrowserRouter>  ← ✅ Proporciona contexto
  <App />
</BrowserRouter>

// App.jsx
<Routes>  ← ✅ ¡Funciona! Ya tiene contexto
  <Route path="/" element={<Home />} />
</Routes>
🧠 ¿Por qué main.jsx es TAN importante? (Click para ver/ocultar)

🚀 main.jsx: El corazón de tu aplicación React

📍 ¿Qué es main.jsx?
  • Punto de entrada: Es el primer archivo que se ejecuta en tu aplicación
  • Conexión con HTML: Conecta tu código React con el HTML del navegador
  • Configuración global: Aquí configuras todo lo que afecta a TODA la aplicación
  • Una sola vez: Solo se ejecuta al cargar la página por primera vez
🔍 Línea por línea explicado:

Línea 1: import { StrictMode } from 'react'

  • StrictMode: Modo estricto que ayuda a detectar problemas en desarrollo
  • Beneficios: Avisos sobre componentes obsoletos, efectos secundarios, etc.
  • Solo en desarrollo: No afecta la versión de producción

Línea 2: import { createRoot } from 'react-dom/client'

  • createRoot: Nueva API de React 18 para renderizar la aplicación
  • react-dom: Librería que conecta React con el DOM del navegador
  • client: Indica que es para el lado del cliente (navegador)

Línea 3: import { BrowserRouter } from 'react-router-dom'

  • BrowserRouter: Habilita la navegación por URL en el navegador
  • Historia del navegador: Permite usar botones de atrás/adelante
  • URLs limpias: Permite URLs como /proyectos en lugar de #/proyectos

Línea 4: import App from './App.jsx'

  • Componente raíz: App es el componente principal de toda la aplicación
  • Todo parte de aquí: Todos los demás componentes están dentro de App

Líneas 6-12: El renderizado

  • createRoot: Crea un "root" React en el elemento HTML con id "root"
  • render: Renderiza todo el árbol de componentes React
  • Jerarquía: StrictMode → BrowserRouter → App → tus componentes
🏗️ ¿Cómo afecta esto a tu día a día?
✅ Cuándo NO necesitas tocar main.jsx:
  • Creando nuevos componentes
  • Agregando nuevas páginas
  • Modificando estilos
  • Trabajando con props y state
  • 95% del desarrollo normal
⚠️ Cuándo SÍ necesitas modificar main.jsx:
  • Agregar Context Providers: Para estado global (como autenticación)
  • Configurar themes: Temas de material-ui, styled-components, etc.
  • Servicios globales: Analytics, error tracking, i18n
  • Configuraciones de librería: Apollo GraphQL, Redux, etc.
🎯 Ejemplo de evolución de main.jsx:

Al principio (como ahora):

StrictMode → BrowserRouter → App

En proyectos más grandes:

StrictMode → BrowserRouter → AuthProvider → ThemeProvider → App
💡 Consejos para el día a día:
  • Déjalo simple: No agregues cosas hasta que las necesites
  • Un lugar, una responsabilidad: Configuraciones globales van aquí
  • Orden importa: Los providers externos van más afuera
  • Documenta cambios: Si agregas algo, deja un comentario

🏆 Conclusión: main.jsx es como el "manager" de tu aplicación - configura el ambiente, pero rara vez necesitas cambiarlo en el desarrollo diario.

📋 Resumen: El Viaje Completo de tu Primer Componente

🎓 ¡Felicidades! Acabas de completar el ciclo completo de React

Quiero que entiendas exactamente lo que acabas de lograr. Has conectado todas las piezas del rompecabezas React:

🏗️ Paso 1: Creaste el componente Header.jsx

📁 Ubicación: src/components/Header.jsx
🎯 Propósito: Una función que devuelve JSX con navegación
💡 Clave: export default Header para que otros archivos lo puedan usar 
function Header() {
  return <header>...</header>;
}
export default Header;

🏠 Paso 2: Lo incorporaste en Home.jsx

📁 Ubicación: src/pages/Home.jsx
🎯 Propósito: Página principal que usa el Header
💡 Clave: import Header + <Header /> 
import Header from "../components/Header";

function Home() {
  return (
    <div>
      <Header />  ← ¡Tu componente funcionando!
      <main>...contenido...</main>
    </div>
  );
}

🗺️ Paso 3: Lo conectaste en App.jsx

📁 Ubicación: src/App.jsx
🎯 Propósito: Define qué página mostrar según la URL
💡 Clave: <Route path="/" element={<Home />} /> 
import Home from "./pages/Home";

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />  ← Home incluye Header
    </Routes>
  );
}

⚡ Paso 4: Le diste vida en main.jsx

📁 Ubicación: src/main.jsx
🎯 Propósito: Punto de entrada que inicia toda la aplicación
💡 Clave: BrowserRouter habilita las rutas + render() muestra todo 
import App from './App.jsx'

createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />  ← App incluye Routes, que incluye Home, que incluye Header
  </BrowserRouter>
)

🎯 El Flujo Completo (de mayor a menor):

main.jsx → renderiza →
  App.jsx → enruta →
    Home.jsx → incluye →
      Header.jsx → ¡SE MUESTRA! 🎉

🧠 ¿Por qué este orden es tan importante?

  • 🧩 Header.jsx: Es el "mueble" - una pieza reutilizable

📝 Nomenclatura en React: Las reglas de oro

🧩 Componentes de React
Nota: El nombre del archivo debe coincidir con el nombre del Componente
  • ✅ Correcto: Header.jsx
  • ✅ Correcto: Tarjeta.jsx
  • ✅ Correcto: Home.jsx
  • ❌ Incorrecto: header.jsx
  • ❌ Incorrecto: tarjeta.jsx
Regla: PascalCase (primera letra mayúscula)
Ejemplo con dos palabras NombreUsuario
Ejemplo con una palabra Usuario Ejemplo de tres palabras TarjetaProductoDestacado
🎨 Archivos CSS
  • ✅ Correcto: Header.css
  • ✅ Correcto: Tarjeta.css
  • ✅ Correcto: Home.css
  • ⚠️ También vale: header.css
  • ⚠️ También vale: tarjeta.css
Regla: Coincide con el nombre del componente
⚡ Variables y funciones
  • ✅ Correcto: const miVariable
  • ✅ Correcto: const productos
  • ✅ Correcto: function miFuncion()
  • ❌ Incorrecto: const MiVariable
  • ❌ Incorrecto: const mi_variable
Regla: camelCase (primera letra minúscula)
💡 Tip del profesor: La consistencia en nomenclatura es clave para trabajar en equipo y mantener código limpio

🤔 ¿No sabes por dónde empezar?

¡Tranquilo! Recuerda el proceso que seguimos con Header:

  1. 📁 Crear: Los archivos Footer.jsx y Footer.css
  2. 📦 Importar: Footer en Home.jsx
  3. 🧩 Usar: <Footer /> en el JSX de Home
  4. 🚀 Probar: Ejecutar npm run dev y ver el resultado

export default Home;
🎯 2. En App.jsx - ¡No necesitas cambiar nada!

App.jsx no necesita modificaciones porque:

  • El Footer está dentro de Home.jsx
  • App.jsx solo se encarga de las rutas
  • Cada página (Home, Proyectos, etc.) puede decidir si incluye Footer o no
📁 3. En main.jsx - ¡Tampoco necesitas cambiar nada!

main.jsx permanece igual porque:

  • Es el punto de entrada de la aplicación
  • Solo configura el router y renderiza App
  • Los componentes específicos se manejan en niveles inferiores

⚠️ Errores Típicos (¡para que no te frustres!)

🚨 Error 1: Nombrar componentes en minúscula

❌ Mal:

function saludo() {  ← minúscula
  return <h1>Hola</h1>;
}

✅ Bien:

function Saludo() {  ← Mayúscula
  return <h1>Hola</h1>;
}

🚨 Error 2: Olvidar className

❌ Mal:

<div class="mi-clase">  ← class

✅ Bien:

<div className="mi-clase">  ← className

🚨 Error 3: Múltiples elementos sin contenedor

❌ Mal:

return (
  <h1>Título</h1>
  <p>Párrafo</p>
);

✅ Bien:

return (
  <>
    <h1>Título</h1>
    <p>Párrafo</p>
  </>
);

💡 Recuerda: Si Visual Studio Code te subraya algo en rojo, ¡prestale atención! Probablemente sea uno de estos errores.
